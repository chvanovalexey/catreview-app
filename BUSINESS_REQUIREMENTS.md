# Бизнес-требования: Система категорийного ревью для торговой сети Дикси

## 1. Общее описание системы

**Назначение**: Корпоративное веб-приложение для торговой сети Дикси управления процессом категорийного ревью, анализа отчётов, отслеживания задач и получения рекомендаций на основе ИИ.

**Целевые пользователи**: Категорийные менеджеры, аналитики, руководители отделов торговой сети Дикси.

---

## 2. Функциональные требования

### 2.1. Аутентификация и безопасность
- Система авторизации с защитой паролем
- Сохранение сессии пользователя
- Защита всех страниц приложения от несанкционированного доступа

### 2.2. Матрица категорийного анализа (главный экран)

**Концепция: От «Библиотеки отчётов» к «Конвейеру Инициатив»**

Матрица построена по единой логике анализа (Super-logic): диагностика → потребности → разрывы → инициативы с прогнозом эффекта.

**Структура матрицы (TO-BE):**
- Двумерная таблица 4×3 (4 строки × 3 столбца)
- **Строки (Рычаги управления, Lever)**: Ассортимент, Цена и Промо, Полка (Merch), Бренды и Поставщики
- **Столбцы (Этапы поиска денег, Super-logic)**:
  1. Здоровье и Эффективность (Internal) — Где мы сейчас?
  2. Потребности и Структура (Shopper) — Что нужно клиенту?
  3. Разрывы с Рынком (External) — Где мы теряем?

**Функциональность ячеек:**
- Каждая ячейка содержит:
  - Бейджи с вопросами, на которые категорийный менеджер может ответить с помощью отчётов
  - Количество отчётов (текущие AS-IS + новые TO-BE)
  - Бейдж с суммой влияния на выручку и маржу (по задачам)
- Клик по ячейке открывает диалог выбора отчёта
- Визуальная индикация:
  - Цветовое кодирование по проценту новых отчётов
  - Анимация появления при загрузке
  - Hover-эффекты

**Типы отчётов:**
- **Текущие отчёты (AS-IS)**: 20 отчётов (REP-01 до REP-20)
  - Ключевые показатели, чековая аналитика, анализ рынка, конкурентов и т.д.
- **Новые отчёты (TO-BE)**: 26 отчётов (NEW-REP-01 до NEW-REP-26)
  - CDT, PPI, KVI Index, Heatmap, NPS, OSA, ABC/XYZ и др.

### 2.3. Диалог выбора отчёта

**Функции:**
- Отображение списка всех отчётов в выбранной ячейке
- Разделение на текущие (AS-IS) и новые (TO-BE) отчёты
- Визуальные бейджи для типов отчётов
- Отображение рекомендации ИИ для ячейки
- Кнопка "ИИ-агент" для открытия чата с агрегированными рекомендациями
- Клик по отчёту открывает детальный просмотр

### 2.4. Просмотр отчёта

**Основные элементы:**
- Встраивание отчёта как веб-компонента из BI-системы Sigla
  - 20 текущих отчётов (AS-IS) уже существуют в Sigla
  - 26 новых отчётов (TO-BE) требуют разработки в Sigla
- Название и ID отчёта
- Навигация "Назад" с сохранением контекста (возврат к диалогу или панели задач)
- Кнопка "ИИ-агент" для получения рекомендаций по конкретному отчёту

**Вкладки:**

1. **Комментарии менеджеров**
   - Дата комментария
   - Автор (ФИО менеджера)
   - Текст комментария
   - Хронологическая сортировка

2. **Задачи по отчёту**
   - Описание задачи
   - Статус: Новая, В работе, Выполнена, Просрочена
   - Срок выполнения
   - Дата создания
   - Влияние на выручку (млн руб)
   - Влияние на маржу (млн руб)
   - Цветовое кодирование по статусу

### 2.5. ИИ-агент (AI Assistant)

**Режимы работы:**

1. **Режим ячейки (агрегация)**
   - Открывается из диалога выбора отчёта
   - Агрегирует рекомендации всех отчётов в ячейке
   - Отображает итоговую рекомендацию с разбивкой по отчётам

2. **Режим отчёта (детальный)**
   - Открывается из просмотра конкретного отчёта
   - Показывает рекомендацию ИИ для конкретного отчёта
   - Персонализированные примеры вопросов

**Функциональность:**
- Отображение рекомендаций ИИ (с форматированием markdown)
- Примеры вопросов (кликабельные для быстрого ввода)
- Чат с ИИ-агентом
- История диалога
- Автоматическая прокрутка к последнему сообщению
- Ввод вопросов через текстовое поле или Enter

### 2.6. Панель задач менеджера

**Основные функции:**
- Боковая панель с полным списком задач
- Фильтрация по статусам: Все, Новая, В работе, Выполнена, Просрочена
- Счётчики задач по статусам

**Агрегированная статистика:**
- Общая выручка по всем задачам (млн руб)
- Общая маржа по всем задачам (млн руб)

**Карточка задачи:**
- Описание
- Статус с цветовым кодированием
- Срок выполнения
- Связанный отчёт (кликабельная ссылка)
- Влияние на выручку
- Влияние на маржу
- Клик по задаче открывает связанный отчёт с сохранением контекста

### 2.7. Навигация и UX

**Контекстная навигация:**
- Сохранение контекста при переходах между экранами
- Возврат к диалогу выбора отчёта при навигации "Назад" из отчёта
- Возврат к панели задач при переходе из задачи

**Адаптивный дизайн:**
- Поддержка desktop и mobile устройств
- Горизонтальная прокрутка матрицы на узких экранах
- Адаптивные диалоги и панели

## 3. Интеграции и ИИ-компонент

### 3.1. Интеграция с BI-системой Sigla

**Встраивание отчётов:**
- Отчёты отображаются как встраиваемые веб-компоненты из Sigla
- Необходима поддержка iframe или Web Components API
- Передача параметров аутентификации для доступа к отчётам
- Обработка событий от встроенных отчётов (опционально)

**Текущие отчёты (AS-IS):**
- 20 отчётов уже существуют в Sigla и готовы к встраиванию

**Новые отчёты (TO-BE):**
- 26 отчётов требуют разработки в Sigla перед интеграцией
- Координация с командой Sigla для создания новых отчётов

### 3.2. ИИ-компонент

**Требования к ИИ-сервису:**
- Генерация рекомендаций на основе данных отчётов
- Два уровня рекомендаций:
  - Агрегированные по ячейке матрицы
  - Детальные по конкретному отчёту
- Генерация примеров вопросов для пользователя
- Чат-бот для ответов на вопросы пользователя
- API для интеграции с frontend

**Данные для ИИ:**
- Исторические данные отчётов из Sigla
- Комментарии менеджеров
- Результаты выполнения задач
- Контекст категории и рынка

### 3.3. Безопасность
- HTTPS для всех соединений
- Защита от XSS, CSRF атак
- Ролевая модель доступа (опционально):
  - Администратор
  - Категорийный менеджер
  - Аналитик (только чтение)
- Логирование всех действий пользователей
- Регулярное резервное копирование данных

## 4. Структура данных

## 5. Дополнительные требования

### 5.1. Масштабируемость
- Возможность добавления новых отчётов без изменения структуры матрицы
- Возможность расширения матрицы (добавление строк/столбцов)
- Поддержка множественных категорий товаров

### 5.2. Интеграции

**Текущие интеграции:**
- **BI-система Sigla**: встраивание отчётов как веб-компонентов
  - 20 текущих отчётов (AS-IS) готовы к интеграции
  - 26 новых отчётов (TO-BE) требуют разработки в Sigla
- **ИИ-сервис**: генерация рекомендаций и чат-бот

- Интеграция с ERP системой Дикси для автоматической синхронизации данных
- Экспорт отчётов в PDF/Excel

- Метрики использования ИИ-агента
